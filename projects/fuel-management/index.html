Fuel Management

A Java/Spring Boot CLI app with authentication/authorization backed by an H2 database, plus a concurrent fueling thread that simulates fueling operations.

Repo
  • Java
  • Spring Boot
  • Concurrency

Demo

Problem

Reacquaint myself with Java by building a small application end-to-end while practicing core OOP principles and making architectural decisions. A key goal was to structure a maintainable codebase and persist data reliably instead of relying on in-memory state.

Solution

  • Used a layered architecture (Controller | Service | Repository)
  • Applied dependency injection with Spring Boot
  • Chose persistence over in-memory state

Code snippet

// This method returns the Fueling Positions that are actively fueling from a table in the database.
@Override
public List<Integer> getActiveFuelingPositionById() {
  return activeTransactionRepo.findAll().stream()
    .map(ActiveTransaction::getFuelingPosition)
    .toList();
}

// This method returns fueling positions that are not currently active, and are available to the equipment whose number has been entered.
@Override
public List<Integer> getInactiveFuelingPositionById(int trId) {
  System.out.println();
  try {
    return fuelingPositionRepo
      .findAll()
      .stream()
      .filter(pr -> startupTransactionRepo.findById(trId).orElseThrow().getEquipmentId().getProducts().stream()
        .map(Product::getProductName)
        .anyMatch(pr2 -> pr.getProduct().getProductName().equals(pr2)))
      .map(FuelingPosition::getFuelingPositionNumber)
      .filter(Predicate.not(hoseN -> activeTransactionRepo.findAll().stream()
        .map(ActiveTransaction::getTransactions)
        .map(Transaction::getFuelingPosition)
        .anyMatch(x -> x.equals(hoseN))))
      .toList();
  } catch (Exception e) {
    System.out.println(e);
    return List.of();
  }
}

What I learned

Coming from mostly C++, this project helped me reacclimate to Java while practicing how to structure an application end-to-end. I rewrote it twice: first to eliminate circular dependencies by tightening responsibilities between layers, and again to rebuild it with Spring Boot's DI and repository patterns. The biggest takeaway was designing around persistence and clear boundaries instead of relying on in-memory state—while also getting hands-on experience with concurrency in a real workflow.
