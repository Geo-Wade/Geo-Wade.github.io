A application that uses allows for authorization and authentication with an H2 database. It then runs a concurrent fueling thread to simulate a fueling process.

Problem

Reacquaint myself with Java by building a small application end-to-end, while practicing core OOP principles and making my own architectural decisions. A key goal was to learn how to structure a maintainable codebase and persist data reliably instead of relying on in-memory state.

Solution

  • Used a layered architecture (Control | Service | Repository)
  • Dependency injection with Spring Boot
  • Persistance over state
